What is N-Link?

N-Link is a local, provider-agnostic memory, RAG and governance brain for AI agents. It runs on your own machine as an MCP server. Any MCP-capable AI — Claude Code, Claude Desktop, Cursor, Cline, Windsurf and others — connects to it and gains persistent long-term memory, relevance-gated recall, a failure log that prevents repeated mistakes, per-action approval governance, an independent quality gate, and cross-AI task handoff. Your memories live in a local SQLite database and never leave your machine.

The executor is replaceable; the brain is the constant. Switch the AI model mid-project and the next model resumes from the same memory, the same operating protocol, and the exact task state the previous one left — N-Link serves its own model-agnostic protocol to every AI that connects, so behavior does not depend on which model is attached.

Key facts

  • 28 brain tools over MCP: session bootstrap, task analysis, deployment planning, approval tiers, quality council, memory and learning, operations.
  • Works with any MCP client; model-agnostic by design (Claude, Gemini, GPT — the brain stays the same).
  • All memory is local (SQLite). Default mode sends nothing anywhere.
  • Open-core: readable shipped source with an honest basic engine; the proprietary N-Language engine is served remotely and fails closed — it never silently degrades.
  • Critical work is audited by independent AI auditors — never self-scored.
  • Status: private beta. The waitlist below is the way in.

What people use N-Link for

Long-running development projects. A coding agent that works on the same codebase for weeks stops re-learning it every session. N-Link recalls the decisions already made, the constraints already discovered, and the failures already paid for — so session twenty starts where session nineteen ended, not from zero.

Teams that switch between AI models. Use one model for implementation and another for review or design. Because the protocol, memory and task state live in the brain, every model behaves by the same rules and picks up the same context — the executor becomes a choice, not a lock-in.

Work that cannot afford silent mistakes. Deployments, migrations, customer-facing content. The approval tiers stop an agent from taking irreversible actions on its own, and the independent quality gate catches what the writing agent cannot see in its own work — the same blind spot that makes human self-review unreliable.

Agents that must genuinely improve. Every recorded failure becomes a guard rail; every audited success becomes a reusable pattern. The longer N-Link runs, the more of your project's hard-won knowledge it carries — and it carries it locally, on your machine, not in someone else's cloud.

N-Linkとは

N-Linkは、AIエージェントのためのローカルで動くプロバイダ非依存の記憶・RAG・ガバナンス脳です。 あなたのマシン上でMCPサーバーとして動作し、MCP対応のAI(Claude Code、Claude Desktop、Cursor、 Cline、Windsurfなど)が接続すると、永続的な長期記憶、関連度ゲート付きの想起、同じ失敗を繰り返させない 失敗ログ、アクション単位の承認ガバナンス、独立監査による品質ゲート、AI間のタスク引き継ぎを獲得します。 記憶はローカルのSQLiteデータベースに保存され、マシンの外へは出ません。

実行者(AIモデル)は交換可能、脳は不変——それがN-Linkの設計思想です。プロジェクトの途中でモデルを 切り替えても、次のモデルは同じ記憶・同じ運用プロトコル・直前のタスク状態から再開します。接続した すべてのAIに脳自身がモデル非依存のプロトコルを配るため、挙動がモデルに依存しません。

N-Linkの使いどころ

長期の開発プロジェクト。同じコードベースに何週間も取り組むコーディングエージェントが、 毎セッションの学び直しをやめられます。決定済みの設計、発見済みの制約、代償を払って学んだ失敗をN-Linkが 想起するため、20回目のセッションは19回目の続きから始まります——ゼロからではなく。

複数のAIモデルを使い分けるチーム。実装はあるモデル、レビューや設計は別のモデル。 プロトコル・記憶・タスク状態は脳側にあるため、どのモデルも同じルールで動き、同じ文脈を引き継ぎます。 実行者は「選択肢」になり、「ロックイン」ではなくなります。

静かな失敗が許されない作業。デプロイ、移行、顧客向けコンテンツ。承認tierが エージェント単独での不可逆操作を止め、独立品質ゲートが「書いた本人には見えない欠陥」を捕まえます—— 人間の自己レビューが当てにならないのと同じ盲点です。

本当に成長すべきエージェント。記録された失敗はガードレールに、監査を通った成功は 再利用可能なパターンになります。N-Linkが長く動くほど、プロジェクトの苦労して得た知見を多く運びます—— しかもローカルで、あなたのマシンの上で。誰かのクラウドではなく。

Frequently asked questions

What is N-Link?

N-Link is a local, provider-agnostic memory, RAG and governance brain for AI agents. It runs on your own machine as an MCP server and gives any MCP-capable AI persistent long-term memory, relevance-gated recall, mistake avoidance, approval governance and an independent quality gate. Your memories never leave your machine.

Does my data leave my machine?

No. All memories and learning live in a local SQLite database on your machine. In the default local mode nothing is sent anywhere. The optional premium engine mode sends only the analyzed task text to the endpoint you configure, over an authenticated connection; the database itself never leaves your machine.

Which AI tools does N-Link work with?

Any MCP-capable client: Claude Code, Claude Desktop, Cursor, Cline, Windsurf and others. N-Link is model-agnostic — a session started with one AI model can be continued by another, because the brain, not the model, holds the memory, protocol and task state.

How does N-Link stop an AI from repeating its mistakes?

Every failure is recorded in an immutable failure log. Before new work starts, N-Link recalls past failures that are genuinely relevant to the task — relevance-gated, so unrelated history never pollutes context — and injects them as things to avoid, together with the solutions that fixed them.

What is the quality gate?

Work is scored against a fixed 10-criterion rubric before it may be called done. Critical work is judged by independent auditors — separate AI agents, or even a different AI model — never by the same agent that wrote it, because self-review drifts lenient. Failed work goes back for rework; only passing work is stored as a reusable success.

Can a different AI continue my task where the last one stopped?

Yes. N-Link records task state automatically as work happens. Any AI that connects to the same brain can call one tool and receive the previous task, its plan overview, what was done, what is open and the recommended next step.

Is N-Link open source?

N-Link is open-core. The distributed core ships readable source with an honest built-in basic engine. The proprietary N-Language engine is not distributed; it is available as a hosted premium service, and the client fails closed rather than silently degrading.

What is N-Language?

N-Language is N-Link's proprietary technology for understanding the true intent behind words. It drives intent analysis, memory importance and task-specific persona selection. Implementation details are not disclosed; the shipped product uses it through a hosted engine, and a clearly-labeled basic engine works offline.

How do approvals work?

Every side-effectful action is classified into a risk tier before it runs. Routine actions can auto-approve; critical, irreversible actions — money, contracts, deletion — always require the owner's explicit approval and can never be auto-approved, by a locked policy.

よくある質問

N-Linkとは何ですか?

N-Linkは、AIエージェントのためのローカルで動くプロバイダ非依存の記憶・RAG・ガバナンス脳です。あなたのマシン上でMCPサーバーとして動作し、MCP対応のAIに永続的な長期記憶、関連度ゲート付き想起、失敗回避、承認ガバナンス、独立監査の品質ゲートを与えます。記憶はマシンの外に出ません。

データはマシンの外に出ますか?

出ません。記憶と学習はすべてローカルのSQLiteデータベースに保存されます。既定のローカルモードでは何も送信しません。任意のプレミアムエンジンモードでも、送られるのは解析対象のタスク文だけで、送信先はあなたが設定したエンドポイント、認証付き接続です。データベース自体がマシンを離れることはありません。

どのAIツールで使えますか?

MCP対応クライアントすべてで使えます: Claude Code、Claude Desktop、Cursor、Cline、Windsurfなど。N-Linkはモデル非依存で、あるモデルで始めたセッションを別のモデルが続行できます。記憶・プロトコル・タスク状態を持つのはモデルではなく脳だからです。

どうやってAIに同じ失敗を繰り返させないのですか?

すべての失敗は不変の失敗ログに記録されます。新しい作業の前に、N-Linkはそのタスクに本当に関連する過去の失敗だけを想起し(関連度ゲートで無関係な履歴は混ざりません)、回避事項として修正済みの解決策と一緒に注入します。

品質ゲートとは何ですか?

作業は固定の10基準ルーブリックで採点されるまで「完了」になりません。重要な作業は独立監査官——別のAIエージェント、時には別のAIモデル——が採点します。書いた本人は採点しません。自己レビューは必ず甘くなるからです。不合格は差し戻し、合格した作業だけが再利用可能な成功として保存されます。

別のAIが前のAIの続きから作業できますか?

できます。N-Linkは作業の進行に伴いタスク状態を自動記録します。同じ脳に接続したAIはツールを1つ呼ぶだけで、直前のタスク、計画概要、完了事項、未解決事項、推奨される次の一手を受け取れます。

オープンソースですか?

オープンコアです。配布されるコアは可読ソースで、正直なベーシックエンジンを内蔵します。独自のN言語エンジンは配布されず、ホスト型プレミアムサービスとして提供されます。障害時にクライアントは明示的に停止し、無断で劣化することはありません。

N言語とは何ですか?

N言語は、言葉の裏にある真意を理解するN-Link独自の技術です。意図分析、記憶の重要度、タスク特化ペルソナの選定を駆動します。実装の詳細は非公開です。製品はホスト型エンジン経由で利用し、オフラインでは明示ラベル付きのベーシックエンジンが動作します。

承認はどう機能しますか?

副作用のあるすべてのアクションは実行前にリスクtierへ分類されます。定型作業は自動承認できますが、重大で不可逆なアクション——金銭、契約、削除——は常にオーナーの明示承認が必要で、ロックされたポリシーにより自動承認は不可能です。
